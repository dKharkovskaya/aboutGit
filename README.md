
# Работа с GIT

1. Команда `pwd` - выводит путь к текущей директории.
2. Команда `cd ~` - переход к домашней директории.
3. Команда `ls` - вывести содержимое директории.
4. Команда `cd имя_папки"` - сменить директорию.
5. Команда `cd ..` - подняться вверх на 1 директорию.
6. Команда `ls -a` - вывести содержимое директории вместе с скрытыми файлами
7. Команда `touch %ИМЯ_ФАЙЛА%` - создать файл.
8. Команда `mkdir %ИМЯ_ДИРЕКТОРИИ%` - создать директорию.
9. Команда `cp index.html src/` - скопировали index.html в папку src.
10. Команда `mv файл папка перемещения` - Синтаксис команды mv аналогичен синтаксису cp. После имени команды указывают список файлов и папок, которые нужно переместить, а затем — папку, в которую нужно выполнить перемещение.
11. Команда `cat %ИМЯ_ФАЙЛА%` - прочитать содержимое текстового файла.
12. Команда `rm %ИМЯ_ФАЙЛА%` - удалить файл.
13. Команда `rmdir %ИМЯ_ДИРЕКТОРИИ%` - удалить пустую директорию.
14. Команда `rm -r %ИМЯ_ДИРЕКТОРИИ%` - удалить директорию со всеми файлами внутри.

**При таком удалении файлы удаляются навсегда, не попадая в корзину**

**Команды в терминале необязательно вбивать и выполнять по очереди. Их можно указывать не по одной, а сразу списком. Для этого их нужно разделить двумя амперсандами (&&).**

15. Команда `git init` - делает папку, в которой сейчас находимся git репозиторием.
16. Команда `rm -rf .git` - приводит к удалению git репозитория в папке.
17. Команда `git status` - показывает текущее состояние репозитория.
18. Команда `git add --all` - добавляет все файлы для остлеживания.
19. Команда `git add %имя_файла%` - добавляет конкретный файл для остлеживания.
20. Команда `git add .` - добавляет всю текущую папку.
21. Команда `git commit -m "Мой первый коммит!"` - делает коммит папки с файлами
22. Команда `git log` - показать историю коммитов.
23. Команда `git remote add origin git@github.com:%ИМЯ_АККАУНТА%/%ИМЯ_ПРОЕКТА%` - привязка удалённого репозитория к локальному. В первый раз используется origin 
24. Команда `git remote -v` - проверка, что репозитории связаны.
25. Команда `git clone %адрес удаленного репозиторию% ` - выкачивает проект с удаленного репозитория на локальную машину.
26. Команда `git push -u origin master` - в первый раз загрузить коммиты в удалённый репозиторий.
27. Команда `git push` - последующая загрузка коммитов в удалённый репозиторий.
28. Команда `java -version` - проверка версии JAVA
29. Команда `javac -encoding UTF-8 %Файл с расширением java%` - компилирование исходного код в байт-код (для 1 файла)
30. Команда `java -Dfile.encoding=UTF-8 %Файл с расширением class%` - компилирование исходного код в байт-код (для 1 файла)
31. Команда `javac -d %Название папки для сохранения% -encoding UTF-8 *.java` - компилирование исходного код в байт-код (несколько файлов)
32. Команда `java -cp bin %Имя класса%` - запуск проекта
33. Команда `jar cfe %имя jar-файла% %имя стартового класса% %список файлов%` - создание архива с расширением jar **jar cfe library.jar Practicum -C bin .**
34. Команда `java -jar %library.jar%` - запуск программы
35. Команда `git commit --amend --no-edit` - Дополнить коммит новыми файлами
36. Команда `git commit --amend -m "Новое сообщение"` - Изменить сообщение коммита
37. Команда `git restore --staged %имя_файла%` - Выполнить unstage изменений
38. Команда `git reset --hard %Идентификатор коммита%` - «Откатить» коммит до определённой ревизии (указанного идентификатора)
39. Команда `git restore %имя_файла%` - Откатить изменения, которые не попали ни в staging, ни в коммит
40. Команда `git diff` - показывает результат сравнения текущей и исходной версии
41. Команда `git log --oneline` - показать краткую информацию о коммитах
42. Команда `git branch` - вывод веток, которые находятся в проекте
43. Команда `git branch %название_ветки%` - создать ветку
44. Команда `git checkout %название_ветки%` - переключиться на нужную ветку
45. Команда `git checkout -b %название_ветки%с - создать ветку и переключиться на неё
46. Команда `git branch -a` - отобразить все ветки
